# 有声书口播制作 Skill

## 核心铁律
**永远先分析文本、制定方案 → 确认后再动手 → 只输出一版，不做多版试听对比。**
**分析时必须了解这本小说属于什么题材，再按题材的演播风格分析。**
**有声喵画本格式时，必须先按画本解析规则处理文本，再做分析和合成。**

---

## Step 0.5：文本解析（两种场景）

### 场景一：有声喵画本格式（有标记）

拿到 DOCX 画本文件时，按以下规则解析：

**角色定义区（不读，只解析）：**
- 格式：`【CV名】【角色名】【性别】【角色描述】【出场数】`
- 用于解析：CV名、角色名、性别、出场数
- 用于角色性格分析（影响对话语气）

**分隔线以下 → 正文：**
- `-----` 分隔线之后的内容都是正文，要读

**章节标题：**
- 格式：`第X章 副标题`
- **只读"第X章"**，副标题不读
- 例如：`第1章 枕边人的毒计（一）` → 只读"第1章"

**• 开头的要点/大纲：**
- 格式：`• 要点内容`
- **要读！** 属于旁白部分，由旁白 CV 负责

**旁白：**
- 没有标记的普通文字
- 由旁白 CV 负责（**每次都要问用户决定旁白分配谁**）
- 语气根据段落内容自动调整，但**相对克制**

**对话：**
- 格式：`【角色名-CV名】"对话内容"`
- **只读引号里的内容**，标记本身不读
- 每个 CV 固定演一个角色，一人一角色，不能一人配多角色
- 语气根据角色性格自动调整（从角色定义区提取性格描述）

**混合行（旁白+对话在同一行）：**
- 例如：`主治医生走过来，面色凝重，【主治医生-一束光的麦】"尚先生，病人的器官衰竭速度……不太正常。"`
- **旁白部分**（标记前）→ 由旁白 CV 读
- **对话部分**（标记后引号内容）→ 由该角色的 CV 读

**一行多角色对话：**
- 例如：`我还没消化完这句话，张姨突然上前，【张姨-有声花小白】"罗……罗小姐上次来探病，偷偷换的。"罗姝，我老婆。`
- 旁白1 → CV旁白
- 【张姨】"罗……罗小姐上次来探病，偷偷换的。" → CV有声花小白
- 旁白2 "罗姝，我老婆。" → CV旁白

### 场景二：纯文本无标记（需要AI分析）

拿到**没有画本标记的纯文本**时，需要先用 AI 分析出哪些是旁白、哪些是对话、有哪些角色，然后让用户来分配 CV。

**分析流程：**

1. **识别章节标题**（如"第X章"）
2. **识别对话内容**：
   - 中文引号：`"对话内容"`、`「对话内容」`
   - 说话动词前缀：`他说：`、`她喊道：`、`医生道：`
3. **识别角色**：
   - 从对话中的称谓和说话动词提取角色名
   - 如：`"尚先生，你的小叔..."` → 角色"尚先生"
4. **标记旁白**：非对话部分即为旁白

**分析结果输出格式（必须让用户确认）：**

```
## 识别到的角色

| 角色 | 性别 | 描述 | 出场次数 |
|:---|:---:|:---|:---:|
| 旁白 | - | 叙述者 | N |
| 尚达达 | 男 | 主角，企业主 | N |
| 罗姝 | 女 | 反派妻子 | N |
| 李锐 | 男 | 主角好友，医生 | N |

## 请分配 CV

请为每个角色指定 CV 名（或由你选择预置音色）：
- 旁白：？
- 尚达达：？
- 罗姝：？
- 李锐：？
- ...
```

**确认后，按分配结果进行合成。**

---

### 旁白分配规则
- **每次都要问用户来决定旁白 CV**
- 旁白语气根据段落内容自动调整，但相对克制
- 除非用户特别说明，旁白不要过于夸张

### 角色分配规则
- 每个 CV 固定演一个角色，一人一角色
- 旁白也是固定的一个人
- 不同 CV 之间声音必须不同（男/女、音色等）
- **纯文本场景下，用户确认角色和 CV 分配后才能开始合成**

## Step 0：题材识别

拿到文本后，**首先判断题材类型**，题材决定了演播的底层风格。

| 题材类型 | 主基调 | 语速 | 节奏特点 |
|:---|:---|:---:|:---|
| **都市现实/写实** | 自然、克制、生活化 | 中偏慢 | 句句实在，不飘不浮 |
| **悬疑惊悚** | 紧张、压迫、神秘 | 偏慢 | 留白多，关键处突然停顿 |
| **言情/情感** | 柔、细腻、代入感 | 中等 | 情绪波动大，快慢交替 |
| **历史/权谋** | 沉稳、厚重、从容 | 偏慢 | 吐字清晰，有分量感 |
| **玄幻/仙侠** | 大气、辽阔、有张力 | 中快 | 场景切换快，节奏多变 |
| **恐怖/灵异** | 阴森、压抑、冷 | 很慢 | 大段留白制造恐惧 |

### 题材专属演播技巧

**都市写实：**
- 真实感第一——不要"播音腔"，要像真人在说话
- 克制比爆发更难——越愤怒越要压着，用沉下去代替吼出来
- 细节决定代入——感官描写要放慢，让听众身临其境
- 段落类型处理：内心独白放慢音量降低、动作描写略快气息略促、感官放慢字字清晰、对话用停顿隔开

**悬疑惊悚：**
- 前期铺垫用低语、慢速、多留白，制造不安感
- 关键线索前放慢加重，停顿[p800]-[p1200]
- 揭示真相时语速突然加快然后突然停止
- 长段落保持单调沉闷，短句突然加速

**言情/情感：**
- 代入感第一——用细腻温柔的语气拉近听众
- 情绪转折时语速变化大：甜蜜加快、心碎放慢
- 独白时降低音量和语速，像心里话
- 对话语气比旁白更生动，有角色感

## Step 1：文本分析

按以下结构输出分析报告：

### 1.1 基调判断
```
题材类型：都市现实 / 悬疑 / 言情 / 历史 / 玄幻 / 恐怖
叙事视角：第一人称 / 第三人称 / 上帝视角
情绪主调：压抑 / 紧张 / 悲伤 / 愤怒 / 平静 / 悬疑 / 温暖 / 激昂
情绪曲线：起点情绪 → 发展 → 高潮 → 回落（标注每个转折点和所在段落）
```

### 1.2 段落类型标注

对每个段落，标注属于以下哪种类型（不同类型有不同处理方式）：

| 段落类型 | 识别特征 | 演播要点 |
|:---|:---|:---|
| 叙事铺垫 | 平铺直叙、背景交代 | 正常语速，沉稳 |
| 动作描写 | 跑、冲、推、扔等动词密集 | 略快、气息略促 |
| 感官描写 | 气味、颜色、声音、触感 | 放慢、字字清晰 |
| 内心独白 | "我想""我知道""我心想" | 放慢、音量降低 |
| 对话 | 引号内容、"他说""她说" | 语气区分，角色感 |
| 情绪爆发 | 突然的情绪变化、震惊 | 前留白，后加重 |
| 冷峻决断 | 决定、下定决心 | 最慢，每个字钉出去 |
| 信息揭露 | 真相、秘密被发现 | 前停顿1秒，放慢加重 |

### 1.3 关键节点标注
- 情绪爆发点（需要放慢加重或留白）
- 悬念/反转处（需要停顿制造效果）
- 信息密度高的段落（需要放慢）

## Step 2：制定演播方案

### 2.1 整体基调设计（必须输出）

```
题材类型：xxx
基调：xxx
语速：中等偏慢/中等/中等偏快（speed=xx-xx）
音量：整体偏x
气息：xxx
音色建议：xxx
```

### 2.2 分段语气设计表（必须输出）

| 段落编号 | 字数 | 段落类型 | 情绪 | 语速 | 停顿处理 |
|:---|:---:|:---|:---|:---:|:---|
| 1 | ~xx字 | 动作描写 | 焦急 | 中快 | 段尾[p600] |
| 2 | ~xx字 | 感官描写 | 沉重 | 慢 | 关键处[p400] |
| ... | | | | | |

### 2.3 停顿标签方案（必须输出）

根据题材和段落类型，选择合适的停顿时长：

| 场景 | 标签 | 适用情况 |
|:---|:---:|:---|
| 句号后自然停顿 | 无（标点自动） | 普通句号 |
| 段落切换 | `[p600]`-`[p800]` | 换段时 |
| 情绪重大转折 | `[p800]`-`[p1000]` | 发现真相、情绪爆发前 |
| 关键信息前 | `[p400]` | 引起注意 |
| 对话前/后 | `[p300]` | 区分叙述和对白 |
| 长句内部 | 人工断句+逗号 | 超过25字切短 |
| 悬念制造 | `[p1000]`-`[p1500]` | 悬疑/恐怖题材专属 |
| 内心独白前 | `[p500]` | 区分叙述和内心 |

## Step 3：引擎选择与合成

### 3.1 引擎优先级
1. **MiMo VoiceDesign**（导演模式，效果最好，无需用户录音）
2. **MiMo VoiceClone**（需要用户录音样本）
3. **讯飞多风格版**（voice_id=assetId, vcn=x6_clone）
4. **讯飞标准版**（voice_id=assetId, vcn=x5_clone）
5. **豆包声音复刻**（speaker=S_xxxxx）

### 3.2 各引擎合成参数

**MiMo VoiceDesign（推荐）：**
```
model: mimo-v2.5-tts-voicedesign
user message: 导演模式（角色+场景+指导）
assistant message: 合成文本（需要分段，每段≤300字）
audio: { format: 'wav', optimize_text_preview: false }
```
- 停顿：通过标点和换行控制
- 语速：在 user message 中用自然语言描述
- 情绪：在 user message 中用自然语言描述
- 输出：24kHz，需要重采样到44.1kHz

**MiMo VoiceClone：**
```
model: mimo-v2.5-tts-voiceclone
user message: 语气指导（自然语言）
assistant message: 合成文本
audio: { voice: data:audio/mpeg;base64,..., format: 'wav' }
```

**讯飞声音复刻（标准版/多风格版）：**
- WebSocket: `ws://cn-huabei-1.xf-yun.com/v1/private/voice_clone`
- 鉴权：header 里放 `app_id`、`status=0`、`res_id`（assetId）
- `parameter.tts.vcn`: 标准版=x5_clone，多风格版=x6_clone
- `parameter.tts.speed`: 0-100（50=正常）
- 停顿：文本中插入 `[p500]` 标签
- 多音字：`着[=zhuo2]手`
- 输出：裸PCM 16bit 24kHz，需要加WAV头

**豆包声音复刻：**
- SSE: `POST https://openspeech.bytedance.com/api/v3/tts/unidirectional/sse`
- `X-Api-Resource-Id`: `seed-icl-2.0`
- `speaker`: S_xxxxx（音色ID）
- 停顿：用标点和换行控制
- 输出：流式MP3/WAV

### 3.3 分段合成策略

| 引擎 | 每段最大字数 | 原因 |
|:---|:---:|:---|
| MiMo VoiceDesign | 300字 | 超过会截断 |
| MiMo VoiceClone | 1200字 | 较宽松 |
| 讯飞声音复刻 | 2000字 | 文本限制 |
| 豆包声音复刻 | 2000字 | API限制 |

### 3.4 输出文件
- 格式：WAV / 44.1kHz / 32-bit float
- 命名：`{章节}_{引擎}_{日期}.wav`
- 目录：`D:\codexvip\audio-outputs\`
  - `voice-clone/doubao/` — 豆包
  - `voice-clone/xfyun/` — 讯飞
  - `voice-clone/mimo/` — MiMo
  - `tts/mimo/` — MiMo VoiceDesign

# Task: 墨境产品经理 Sprint 1 — 需求梳理与迭代规划

## Goal
作为产品经理，基于当前项目状态（已完成清单 + 待完成清单），输出结构化的迭代计划需求文档，并优先推动 P0/P1 级别任务的执行。

## Scope
- 分析待完成清单，按 RICE + MoSCoW 方法重排优先级
- 为 P0/P1 任务输出完整的任务文档（用户故事、功能拆解、验收标准、涉及角色、预估工作量）
- 制作 Sprint 1 迭代计划
- 为每个待办空壳功能输出完整需求

## Non-goals
- 不直接写代码
- 不做 UI 设计
- 不干预开发团队的具体实现

## Success Criteria
- 输出结构化的 Sprint 1 迭代计划文档
- 每个 P0/P1 任务输出标准需求文档
- 空壳功能全部输出完整需求

## Priority Key (MoSCoW)
- P0=Must 不解决就没法用
- P1=Should 重要但不阻塞
- P2=Could 有价值但可等
- P3=Won't 本期不做

## Verification Gates
- 每个需求文档包含完整 5 要素（用户故事、功能拆解、验收标准、涉及角色、预估工作量）
- 优先级有明确的 RICE 评分依据

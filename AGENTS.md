# AGENTS.md — Codex Workspace 协作规则

## 全局执行纪律

### 技能包使用
- 当任务匹配已安装技能包的 `description` 时，必须触发该技能包
- 触发后必须严格遵循其 SKILL.md 定义的工作流、工具链和流程
- 禁止在技能包触发后用自己的直觉替代技能包定义的执行方式
- 如果技能包提供了脚本（如 `with_server.py`），必须使用该脚本而非手动执行等价操作

### 诊断流程规范
任何故障诊断、验证测试任务，必须走以下流程：
1. 先诊断分析原因
2. 出修复方案
3. 等张总确认方案
4. 执行修复
5. 用技能包规定的验证方式做完整验证

禁止在未出方案和未确认前直接动手修复。

### 墨境项目
- 故障诊断必须触发 `mojing-diagnose` 技能包
- 严格遵循该技能包的 SKILL.md 流程

### 如果发现我（Codex）没有遵守以上纪律
- 张总：直接说"你读一下AGENTS.md"
- 我会重新加载并纠正行为

### Git 提交纪律（跨会话、跨项目通用）

每次 AI 完成一轮代码改动后，必须立即执行：

```bash
git add src/ public/ package.json next.config.ts 等改动文件
git commit -m "type: 做了什么"
```

**核心原则：**
- **做完就提交**，不让改动留在工作区或 stash 中
- commit message 写清楚做了什么，方便后续 AI 通过 `git log` 了解项目状态
- 如果改动涉及多个独立功能，拆成多个提交
- 禁止跨多个会话累积未提交的改动

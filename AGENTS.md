# AGENTS.md — Codex Workspace 协作规则

## 全局执行纪律

### 技能包使用
- 当任务匹配已安装技能包的 `description` 时，必须触发该技能包
- 触发后必须严格遵循其 SKILL.md 定义的工作流、工具链和流程
- 禁止在技能包触发后用自己的直觉替代技能包定义的执行方式
- 如果技能包提供了脚本（如 `with_server.py`），必须使用该脚本而非手动执行等价操作

### 诊断流程规范
任何故障诊断、验证测试任务，必须走以下流程：
1. 先诊断分析原因
2. 出修复方案
3. 等张总确认方案
4. 执行修复
5. 用技能包规定的验证方式做完整验证

禁止在未出方案和未确认前直接动手修复。

### API 调用铁律（跨项目通用，永远不能违反）

> **🔴 任何消耗 API 额度、费用、资源的操作，在张总明确说"可以"之前，绝对不允许执行。**
> **说"听听效果"是让听已有的文件，不是让继续合成。**
> **违反这条铁律，就是烧张总的钱。永远不能再犯。**

包括但不限于：
- 调用任何语音合成 API（豆包/讯飞/MiMo）
- 调用任何 AI 模型 API（PaddleOCR/千问/文心等）
- 执行任何可能产生费用的操作

**正确流程：**
```
1. 设计方案 → 输出方案内容（不执行）
2. 问张总："方案如上，可以执行吗？"
3. 等张总明确说"可以"或"试试" → 才能动手
4. 如果张总说"听听效果"、"我听听"等 → 这是让张总去听已有的文件，不是让继续合成
```

**违反这条铁律，就是烧张总的钱。永远不能再犯。**

### Git 提交纪律（跨会话、跨项目通用）

每次 AI 完成一轮代码改动后，必须立即执行：

```bash
git add src/ public/ package.json next.config.ts 等改动文件
git commit -m "type: 做了什么"
```

**核心原则：**
- **做完就提交**，不让改动留在工作区或 stash 中
- commit message 写清楚做了什么，方便后续 AI 通过 `git log` 了解项目状态
- 如果改动涉及多个独立功能，拆成多个提交
- 禁止跨多个会话累积未提交的改动

## 什么是技能包？

技能包是 AI 助手的"专业执照"。装上一个技能包，AI 就从一个啥都会一点的通用助手，变成一个**懂行规、懂流程、懂细节的专家**。

比如：
- 你跟它说"帮我配一段有声书" → 它知道先分析文本、设计语气、选参数、合成、检查静音间距、加淡入淡出……
- 你跟它说"帮我做个海报" → 它自动走设计流程，输出不像模板的高端作品
- 你跟它说"代码审查一下" → 它知道看哪些维度、怎么给有建设性的反馈

**不用技能包：** AI 乱猜流程，漏细节，出活质量看运气。
**用了技能包：** AI 按标准流程走，每一步都在点上，结果稳定可预期。

## 一、有声书制作（AI 配音导演）

### 技能：`aubook-narration-producer`

**一句话：** 给 AI 装上 10 年经验的配音导演大脑，能把一本小说变成专业级有声书。

**它能干的事：**

| 你只要说 | AI 自动完成 |
|:---|:---|
| "这段小说帮我配成有声书" | 分析题材→设计语气→调参数→合成→加淡入淡出 |
| "这里语气要压抑一点" | 自动调整 context_texts 表演指令 |
| "这段旁白和对话交替出现" | 启动编排脚本，自动对齐拼接+打AU标记 |
| "加个润色效果" | 调用 pedalboard 压缩+混响（多预设可选） |

**为什么它做得好：**
- **cot标签技术**：能对每一句话单独控制语速和语气，不是整段一个调
- **三层上下文设计**：告诉 AI "前情摘要+剧本笔记+表演指导"，像导演给演员说戏
- **多轨编排**：旁白和角色对话交替时，自动计算间距、拼接、打标记
- **效果链预设**：润色/电台/空旷/低沉四种效果可选，不覆盖原文件

**适合谁用：** 有声书制作人、配音工作室、自媒体创作者、小说作者想做有声版

**怎么用：**
```
你："帮我把这段画本做成有声书"
AI：自动按流程执行
```

## 二、防止 AI 过度编程（懒人模式）

### 技能：`ponytail`

**一句话：** 专治 AI "为了加一行功能装十个库"的毛病。

**它能干的事：**

| 级别 | 效果 | 怎么说 |
|:---|:---|:---|
| **lite** | 完成任务，顺带提一句更懒的方案 | `/ponytail lite` |
| **full** | 强制执行"能不写就不写"原则 | `/ponytail` |
| **ultra** | 极端精简，能删就删 | `/ponytail ultra` |
| **关闭** | 恢复正常模式 | `/ponytail off` |

**核心逻辑（决策阶梯）：**
```
写代码前 AI 会自问：
1. 这个功能真的需要吗？    → 不需要就不做
2. 代码库里已经有了吗？    → 有就复用
3. Python 标准库能做吗？   → 用标准库
4. 浏览器原生支持吗？      → <input type="date"> 比装日期库好
5. 已有的依赖能做到吗？    → 不为了几行代码加新依赖
6. 一行能搞定吗？          → 一行
7. 都不行 → 写最小可用的
```

**效果（实测数据）：** 代码量减少 54%，token 省 22%，成本降 20%，耗时快 27%。

**适合谁用：** 任何让 AI 写代码的人——程序员、创业者、学生

**怎么用：**
```
你："/ponytail 帮我写个日期选择组件"
AI：直接用 <input type="date">，不装 flatpickr，不写 wrapper
```

## 三、语音工具集成

### 技能：`voice-toolkit`

**一句话：** 语音相关的杂活——识别图片文字、克隆声音、合成语音——一个技能包全搞定。

**它能干的事：**
- 图片/文档里提取文字（PaddleOCR / 腾讯混元）
- 声音克隆（几秒样本就能复刻音色）
- 语音合成（豆包 / 讯飞 / MiMo）
- 画本解析、有声书演播

**适合谁用：** 需要批量处理音频/图片文字的用户

# 有声书AI配音工作流 — 完整技术说明

> 本文档供开发可视化配音网站的AI参考。
> 完整描述了从画本到成品音频的全套工作流、技能包、脚本工具和参数体系。

---

## 一、整体流程概览

```
画本（.txt）
  │
  ├─ Step 0.5 文本解析 ──→ 分离旁白和角色对话
  ├─ Step 0    题材识别 ──→ 决定演播底层风格
  ├─ Step 1    文本分析 ──→ 情绪曲线、段落类型、关键节点
  ├─ Step 2    演播方案 ──→ context_texts三层设计 + cot标签
  ├─ Step 3    引擎配置 ──→ 豆包Expressive参数
  ├─ Step 4    执行合成 ──→ 调API、段间淡入淡出
  ├─ Step 4.5  多轨编排（可选）──→ 自动拼接+打标
  └─ Step 5    后处理（可选）──→ pedalboard润色
```

## 二、核心技术体系

### 2.1 语音合成引擎

**主力引擎：豆包 ICL2.0 Expressive（seed-tts-2.0-expressive）**

这是字节跳动的云端语音合成大模型，支持精细的表演控制。

| 参数 | 说明 | 默认值 |
|:---|:---|:---:|
| `model` | 模型版本 | `seed-tts-2.0-expressive` |
| `voice_type` | 音色ID | `zh_female_meilinvyou_moon_bigtts` 等 |
| `speech_rate` | 语速 | -50 ~ 100（0=正常） |
| `pitch` | 音调 | -12 ~ 12（0=正常） |
| `loudness_rate` | 音量 | -50 ~ 100（0=正常） |
| `format` | 输出格式 | mp3 / wav |
| `sample_rate` | 采样率 | 24000 |

**关键参数（表演控制）：**
- `context_texts`：**语音指令数组**，告诉引擎用什么情绪和语气朗读
- `use_tag_parser`：是否开启 **cot标签** 功能，设为 `true`
- **cot标签**：在文本中嵌入 `<cot text=描述>内容</cot>` 来控制每句话的语速/语气

**备选引擎：** 讯飞多风格版（`x6_clone`，支持rhy情感参数）/ MiMo VoiceDesign

### 2.2 表演控制体系（核心竞争力）

#### context_texts 三层设计法

这是有声书配音的核心方法论。每一段旁白合成时，需要提供三层信息：

```
第一层：前情摘要
  → 上文发生了什么？主角当前心理状态？
  → 例："主人公出差回来刚下飞机，看到十几个未接来电"

第二层：剧本笔记
  → 这段在情绪弧线上的作用？承接/推进/转折/爆发？
  → 例："得知小叔病危的噩耗，从震惊到接受"

第三层：表演指导
  → 具体用什么语气、情绪、语速、场景？
  → 例："用低沉压抑的语气，带着刚得知噩耗后的震惊与难以置信，语速由平缓突然加快"
```

三层信息合并成一段自然语言描述，写入 `context_texts` 数组中。

#### cot标签（单句级控制）

在文本中嵌入标签，实现对**每一句话**的精确控制：

```
<cot text=语速放慢>他沉默了很久。</cot>
<cot text=语速加快、呼吸局促>然后猛地站了起来。</cot>
```

**铁律：旁白必须全部加 `全程保持匀速`**，特殊情绪处也只能微调（"放慢一点点"、"加快一点点"），不能大幅变化。角色音则不受限制，可根据角色情绪自由变化。

### 2.3 音频拼接规则

| 规则 | 值 |
|:---|:---:|
| 段间淡入淡出 | ≥80ms |
| 夹角色音处静音 | ≥800ms |
| 输出格式 | WAV（44.1kHz / 16bit / 单声道→立体声） |
| 命名格式 | `{书名}-{集数:02d}-{CV名}-旁白.mp3` |

## 三、技能包（Skill）体系

当用户提到"有声书/旁白/口播/语音合成/画本"时，AI 会自动触发 `aubook-narration-producer` 技能包，按以下流程执行：

### Step 0.5：文本解析
输入画本（.txt 格式），按正则规则分离旁白和角色对话：
- 去掉【角色-CV】"对话内容" → 得到纯旁白
- 记录每个角色对话的位置 → 生成打标信息

### Step 0：题材识别
判断题材类型（都市/古风/悬疑/言情等），决定演播底层风格。

### Step 1：文本分析
通读画本，分析：
- 情绪曲线（起点→铺垫→发展→转折→高潮→回落）
- 段落类型标注（叙事/描写/对话/内心独白）
- 夹角色音位置

### Step 2：演播方案
为每段设计 context_texts（三层）+ cot 标签。

### Step 3：引擎配置
配置豆包Expressive参数。

### Step 4：执行合成
每段独立调 API 合成，段间 80ms 淡入淡出 + 1秒静音。

### Step 4.5（可选）：多轨编排合成
使用编排脚本自动处理多角色对话位（详见第四节）。

### Step 5（可选）：后处理润色
使用 pedalboard（Spotify 开源音频处理库）做压缩器和混响效果。

## 四、多轨编排脚本（本次新增）

### 4.1 解决了什么问题

传统流程中，画本的旁白和角色对话交替出现时，需要手动数着时间点去对齐拼接。编排脚本自动化了这个过程。

### 4.2 工作流程

```
画本.txt
  → 解析（区分旁白和对话位）
  → 编排（生成带静音间距的清单）
  → 批量合成（调用豆包API）
  → 拼接（旁白+静音+旁白+...）
  → 打标（写入AU cue标记）
  → 润色（可选）
```

### 4.3 输入格式

**模式A（自动解析）：** 标准【角色-CV】"对话"格式
```text
【旁白-墨染青衣Nicou】"叙述文字..."
【魏总-墨染青衣Nicou】"对话内容..."
```

**模式B（手动标注）：** 精确控制段落边界
```text
|旁白_START|
叙述文字...
|旁白_END|
|角色_标记01|
```

### 4.4 编排清单结构

脚本将画本解析为三种段类型：

| 段类型 | 说明 |
|:---|:---|
| `narration` | 旁白文本，需要合成语音 |
| `dialog_marker` | 对话标记位，插入静音并打标 |
| `silence` | 静音段，默认≥800ms |

### 4.5 合成任务管理

- 生成 `synthesize_tasks.json` 任务清单
- 支持断点续传（已合成的段自动跳过）
- 合成前必须输出清单让用户确认（防止未经确认调API）

### 4.6 AU标记写入

在拼接后的 WAV 文件中写入 cue + LIST/adtl 块，Adobe Audition 打开即可看到标记点，标记命名格式为"标记01"、"标记02"...

### 4.7 代码架构

```
scripts/
├── narrate-arrange.py          # 主入口（CLI）
└── narrate_arrange_lib/
    ├── parser.py               # 画本解析器（两种模式）
    ├── arranger.py             # 编排清单生成
    ├── synthesizer.py          # 合成任务管理
    ├── concatenator.py         # 音频拼接+淡入淡出
    ├── marker.py               # AU cue标记写入
    └── effects.py              # pedalboard后处理
```

### 4.8 命令行参数

```bash
# 预览编排清单（不调API）
python narrate-arrange.py 画本.txt --dry-run --book 书名 --episode 01 --cv 配音员名

# 完整执行
python narrate-arrange.py 画本.txt --book 书名 --episode 01 --cv 配音员名

# 手动标注模式
python narrate-arrange.py 画本.txt --mode annotated --book 书名

# 自定义静音时长
python narrate-arrange.py 画本.txt --silence-ms 1000
```

## 五、执行纪律（网站设计时需遵守）

### API调用铁律（最重要）
- **任何消耗 API 额度/费用的操作，必须等用户明确说"可以"才能执行**
- 合成前必须输出清单让用户确认
- 用户说"听听效果"是指听已有的文件，不是继续合成

### 流程铁律
- 不能跳步：必须按 Step 0→1→2→3→4 顺序执行
- 每一步做完输出文档，等用户说"可以"才能进下一步
- 不能合并步骤

### 间隔铁律
- 所有夹角色音位置必须全部处理，少一处都不行
- 每处间隔必须 ≥800ms
- context_texts 必须包含三层（前情+笔记+指导），少一层都不行
- cot 标签必须加"全程保持匀速"（旁白）

## 六、网站需提供的核心功能

1. **画本上传/粘贴** → 支持 .txt 格式
2. **AI文本分析** → 自动识别题材、情绪曲线、段落类型
3. **context_texts编辑** → 为每段编写三层语音指令，AI辅助生成
4. **cot标签编辑器** → 可视化标记每句话的语速/语气
5. **引擎参数面板** → 音色选择、语速/音调调节
6. **编排清单预览** → 显示旁白段+对话标记位+静音间距
7. **合成任务管理** → 任务列表、合成进度、断点续传
8. **音频预览** → 试听单段/整章
9. **后处理选项** → 压缩/混响开关
10. **输出下载** → WAV/MP3 带标记或不带标记

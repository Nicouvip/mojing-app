# Phase 2：实现

> **什么时候用：** 任务拆好了，要开始写代码了。
> **通俗说：** 终于开始干活了。

---

## 做什么

```
开新对话 → 读文档 → 挂技能 → 写代码 → 子代理并行
```

---

## 步骤

### [2.1] 启动新对话

**目的：** 每个功能开一个独立的聊天，防止上下文太长。

**张总需要做的：** 新对话里说 **"加载墨境配置"** 就行

**我会做：**
- 自动加载上下文（规则、技能路由表）
- 读对应的设计文档
- 挂对应的 SKILL.md（不挂技能不开工）

### [2.2] 选实现策略

**目的：** 根据要做的内容，用最合适的工具。

**我会判断：**
- **逻辑复杂、正确性很重要** → 挂 `tdd` 技能（先写测试再写代码，确保不出错）
- **改界面/页面** → 挂 `frontend-ui-engineering` 技能
- **改接口/API** → 挂 `api-and-interface-design` 技能
- **复杂的** → 同时挂多个技能

### [2.3] 执行实现

**目的：** 真正开始写代码。

**做法：**
- 按拆好的任务顺序一步一步做
- 每个任务改完立刻验证一次
- 需要同时干的 → 派子代理并行执行
  - `general` 子代理 → 干活的
  - `explore` 子代理 → 查资料、研究问题
  - `design-reviewer` 子代理 → 随时审查
- 每完成一步，更新进度

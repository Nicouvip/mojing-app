# Phase 5：运维与迭代

> **什么时候用：** 功能已经上线了，要维护、改进。
> **通俗说：** 上线不是结束，用起来才是开始。

---

## 做什么

```
自动化运维 → 反馈 → 修复 → 知识沉淀
```

---

## 步骤

### [5.1] 自动化运维

**目的：** 重复的事情让机器自动做，不用每次都手动。

**做法：**
- **每天要做的** → 设定时任务 `gui_schedule_create (daily)`
  - 比如：每天自动检查编译有没有报错
- **每隔一段时间要做的** → 设间隔任务 `gui_schedule_create (interval)`
  - 比如：每 30 分钟检查一次服务状态
- **某个时间点要做一次** → 设一次性任务 `gui_schedule_create (at)`
  - 比如：下周一提醒张总验收
- **重复做了 3 次以上的流程** → 做成 workflow 保存起来
  - 以后只需要说"运行XX流程"就行

### [5.2] 反馈闭环

**目的：** 张总用了发现问题 → 修复 → 再给张总看。

**流程：**
```
张总反馈问题
    ↓
我确认理解了
    ↓
我分析原因
    ↓
我出修复方案
    ↓
张总确认方案
    ↓
走 Phase 1-4 修复流程
    ↓
张总再看效果
```

### [5.3] 知识沉淀

**目的：** 做过的东西不白做，变成以后的资产。

**做法：**
- **重要决策** → 记到项目文档里，以后不会忘
- **反复做的流程** → 做成 workflow，一键执行
- **重复 3 次以上的操作** → 做成配方或技能
- **技术债务** → 记录下来哪些地方以后需要优化

# Phase 1：任务规划

> **什么时候用：** 方案已确定，需要拆成具体任务来干活。
> **通俗说：** 把"要做的东西"切成一块一块能下手干的。

---

## 做什么

```
设计文档 → Git分支 → 拆任务 → 标依赖 → 标并行
```

---

## 步骤

### [1.1] 开 Git 分支

**目的：** 每个功能单独开一条线来开发，互不干扰。

**做法：**
- `git checkout -b feature/功能名称`
- 分支名用英文：比如 `feature/compliance-panel`、`feature/cooling-matrix`

**为什么要这么做：**
- 主分支（master）一直是稳定可用的
- 开发中的代码不会影响其他功能
- 做一半不想做了可以直接扔掉分支，不影响主代码

### [1.2] 拆任务

**目的：** 把一个大功能切成小步骤，一步一步走。

**做法：**
- **简单的（3步以内）** → 我直接干，不用拆
- **复杂的（多于3步）** → 我挂 `planning-and-task-breakdown` 技能

**拆任务的原则：**
- 每个任务都是"做一个完整的东西"（不是"先做数据库、再做界面"）
- 比如：任务A="用户能注册"、任务B="用户能登录"（每个都是完整功能）
- 用 `task_graph` 建依赖关系（A做完了才能做B）

### [1.3] 标出能同时干的

**目的：** 没有依赖的任务可以同时做，节省时间。

**做法：**
- 我看看哪些任务之间没有依赖关系
- 标记它们为"可并行"
- 等执行阶段派子代理同时开干

---

## 张总只需要做的

1. 看任务拆得合不合理
2. 说"行，开干"

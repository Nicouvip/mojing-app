# AI请求重试机制

## 变更文件

**src/lib/utils/fetch-with-timeout.ts**

原文件仅包含简单 `fetch` + `AbortController` 超时，无重试逻辑。已增强为带指数退避重试的完整封装。

## 改动说明

### 1. 新增 `FetchRetryError` 类

```typescript
export class FetchRetryError extends Error {
  retryCount: number
  status?: number
}
```

- 携带 `retryCount`（第几次重试失败）和 `status`（HTTP状态码）
- 各 route.ts 可直接 catch 此异常判断重试情况

### 2. 新增 `RetryOptions` 接口

```typescript
export interface RetryOptions {
  maxRetries?: number   // 默认3
  onRetry?: (attempt: number, total: number) => void
}
```

- `onRetry` 回调允许调用方感知每次重试（可用于日志/UI提示）
- 所有参数可选，保持向后兼容

### 3. `fetchWithTimeout` 重试逻辑

| 场景 | 行为 |
|------|------|
| 2xx/3xx 成功 | 立即返回 Response |
| 4xx 客户端错误 | 不重试，立即返回 Response |
| 5xx 服务端错误 | 重试（最多3次） |
| 网络错误（超时/DNS失败/连接拒绝） | 重试（最多3次） |
| 所有重试耗尽 | 抛 `FetchRetryError` |

**退避策略：** 1s → 2s → 4s（`Math.pow(2, attempt - 1) * 1000`）

### 4. 不改各 route.ts

函数签名新增第4个可选参数 `retryOptions?: RetryOptions`，所有现有调用保持原样，自动获得3次重试能力。

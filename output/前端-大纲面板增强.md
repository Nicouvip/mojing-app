# 大纲面板增强

## 改动文件

`src/app/editor/[id]/page.tsx`（仅此一个文件）

## 改动内容

将右侧面板「📑 大纲」tab 的标题列表从只读/悬停交互改为**始终可见的可交互面板**。

### 具体变更

| 需求 | 实现方式 |
|------|----------|
| ① 可交互标题列表 | 每个标题项渲染为可点击行，hover 有背景反馈 |
| ② 层级缩进 | `paddingLeft: `${(h.level - 1) * 16 + 4}px`` — h1 无缩进，h2 缩进 16px，h3 缩进 32px |
| ③ ↑↓ 箭头调整顺序 | 每个标题右侧始终显示 ↑↓ 按钮，调用 `swapHeadings(posA, posB)` 交换两标题文字 |
| ④ 点击跳转到对应段落 | 标题文本区域点击调用 `scrollToHeading(pos)` 聚焦并滚动到编辑器对应位置 |
| ⑤ ✏️ 内联编辑文字 | 点击 ✏️ 进入内联输入模式，Enter/Blur 保存（调用 `updateHeadingText`），Escape 取消 |
| ⑥ ❌ 从大纲移除 | 点击 ❌ 调用 `deleteHeadingAt(pos)` 删除编辑器中的标题节点，不影响其他内容 |

### 代码要点

- **删除**了 `group` 类和 `opacity-0 group-hover:opacity-100` 隐藏逻辑，使按钮始终可见
- **移动**了 ↑↓ 按钮位置：从标题左侧移到右侧，与 ✏️ ❌ 合并成一组操作按钮
- 按钮默认使用 `text-muted-foreground/50` 半透明，hover 时变为 `text-foreground`（❌ 为 `text-destructive`）
- 所有操作均通过 `writingEditorRef.current` 调用 WritingEditor 组件暴露的 `EditorHandle` 方法

# 前端 — 卷管理 + 字数目标

## 改了什么

### 任务1（P1）：卷管理完善（重命名/删除）

**3 个文件修改：**

| 文件 | 改动量 | 说明 |
|------|:------:|------|
| `src/lib/db/types.ts` | +11 行 | 新增 `Volume` 接口（id/projectId/name/order/createdAt/updatedAt）；`Chapter` 新增 `volumeId?: string` |
| `src/lib/db/store.ts` | +81 行 | 新增卷存储 + 5 个函数：`getVolumes`、`createVolume`、`renameVolume`、`deleteVolume`、`ensureDefaultVolume`；`createProject`/`createChapter` 适配默认卷和 volumeId |
| `src/app/editor/[id]/page.tsx` | ~120 行 | 卷 UI 全面改造 |

**具体改动：**

1. **类型（types.ts）**
   - 新增 `Volume` 接口
   - `Chapter.volumeId` 可选字段，空字符串表示未分类

2. **存储层（store.ts）**
   - `memVolumes` + localStorage `mojing_volumes` 持久化
   - `getVolumes(projectId)` — 按 order 排序返回
   - `createVolume(projectId, name)` — 自动生成 id 和 order
   - `renameVolume(volumeId, name)` — name 为空不修改
   - `deleteVolume(volumeId)` — 章节移入未分类（volumeId 置空），返回章节数
   - `ensureDefaultVolume(projectId)` — 无卷时创建默认"第一卷"
   - `createProject` 创建时自动附带默认卷
   - `createChapter` 新增可选 `volumeId` 参数，默认归入第一卷

3. **UI（page.tsx）**
   - 卷状态从 `string[]` 改为 `Volume[]`
   - 加载时兼容旧 `mojing_volumes_{projectId}` localStorage 字符串数组格式，自动迁移
   - 章节按 `volumeId` 分组展示（替代旧等分公式）
   - 无 `volumeId` 的章节显示在"📂 未分类"分组
   - 卷右侧"···"下拉菜单：
     - ✏️ 重命名 — 内联输入框，回车确认 / Esc 取消
     - ❌ 删除卷 — 空卷直接删；非空卷弹窗提示"卷内有 N 篇章节，删除后章节将移入未分类"
   - 新建章默认归入第一卷
   - 新建卷改走 `createVolume`

### 任务2（P2）：字数目标手动输入

**零改动。** 该功能已在 `src/components/writing-editor.tsx` 中实现：
- `<input type="number">` 手动输入（Enter 确认 / Blur 确认）
- `+500` 快捷按钮
- 实时字数显示 `{wc}/{wordGoal}字`
- 进度条

## 为什么改

- 任务指令 TASK-B4：卷管理功能严重残缺，alert('卷功能待实现') 占位
- 章节非用户自定义分配到卷（旧版按等分公式自动分配）
- 无法重命名 / 删除卷
- 字数目标：只有 +500 按钮，缺少手动输入入口

## 对外影响

| 影响范围 | 说明 |
|---------|------|
| types.ts | Volume 接口新增，Chapter 新增 volumeId，影响 store.ts 和 page.tsx |
| store.ts | 新增卷 CRUD 函数，不破坏旧接口 |
| page.tsx | 卷 UI 重写，旧 `mojing_volumes_{projectId}` localStorage 数据自动迁移 |
| localStorage | 新增 `mojing_volumes` 存储所有 Volume 对象（统一 key）；旧 key 迁移后自动删除 |
| 无破坏性变更 | 旧项目打开后自动创建默认卷，旧章节自动归入未分类 |

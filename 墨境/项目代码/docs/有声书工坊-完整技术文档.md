### 1.2 核心设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| TTS引擎 | MiMo V2.5 (小米) | 8款中文预置音色 + 情绪控制 + 风格标签 |
| 文本分析 | DeepSeek V4 Flash | 中文理解强、速度快、成本低 |
| 音频格式 | WAV (PCM16 24kHz) | 无损，方便后处理 |
| 状态管理 | React useState + localStorage | 有声书数据暂不持久化到数据库 |
| 段落编辑 | textarea contenteditable | 支持直接编辑段落文本 |
| 撤回/重做 | 内存栈 (最多30层) | 轻量级，无需外部库 |

## 4. API 路由详解

### 4.1 POST /api/audiobook/analyze

**功能：** AI分析小说文本，识别角色、情绪、音色推荐

| 项目 | 详情 |
|------|------|
| 引擎 | DeepSeek V4 Flash（直接fetch） |
| 超时 | 120秒 |
| 文本限制 | 超过15000字符会截断 |
| 重试 | JSON解析失败最多3次重试 |

```
Request:  { text: string }
Response: {
  success: boolean,
  characters: CharacterAnalysis[],
  segments: SegmentAnalysis[],
  narrationStyle: NarrationStyle,
  meta: { totalSegments, dialogueCount, narrationCount, characterCount }
}
```

**处理流程：**
1. `buildAnalysisPrompt(text)` 构建提示词
2. 调用 DeepSeek `deepseek-v4-flash`
3. 提取JSON（处理markdown代码块包裹）
4. 校验结构（segments数组 + characters数组）
5. 前端调用 `processAnalysisResult()` 合并优化

### 4.2 POST /api/audiobook/generate

**功能：** 单段TTS语音合成

```
Request:  {
  text: string,
  voice: string,
  emotion?: string,
  style?: string,
  format?: 'wav' | 'mp3',
  emotionIntensity?: number,  // 1-10
  speed?: 'slow' | 'normal' | 'fast',
  specialNote?: string
}
Response: {
  success: boolean,
  audio: string,   // base64
  duration: number,
  format: string
}
```

**关键实现：** `MiMoTTSEngine.generate()` 内部构建风格指导文本：
```
styleParts = [emotion, "情感强度X/10", "语速缓慢/加快", specialNote, style]
→ 聚合为 user message → text 作为 assistant message → MiMo API
```

### 4.3 POST /api/audiobook/import

**功能：** 小说文本导入 + 智能分章

```
Request:  {
  projectId: string,
  text: string,
  title?: string,
  splitMode?: 'auto' | 'manual' | 'none'
}
Response: {
  success: boolean,
  projectId: string,
  chapters: [{ title, content, wordCount }],
  stats: { totalChapters, totalWords }
}
```

**分章逻辑：**
- `auto`: 正则匹配「第X章」「Chapter X」等 → 失败则按空行分
- `manual`: 按连续空行分
- `none`: 不分，整体为一章

### 4.4 POST /api/audiobook/import-book

**功能：** 画本导入（TXT/JSON格式）

```
Request:  { format: 'txt' | 'json', content: string, chapterId?: string }
Response: {
  success, title?, characters: CharacterAnalysis[],
  segments: SegmentAnalysis[], meta
}
```

**TXT格式：**
```
## 角色表
角色名：性别，年龄，性格描述
## 画本内容
序号 | 角色名 | 情绪 | 文本内容
```

**JSON格式：**
```json
{ "title": "...", "characters": {...}, "lines": [{ "type", "character", "text", "emotion" }] }
```

### 4.5 POST /api/audiobook/import-docx

**功能：** DOCX画本导入（有声喵格式）

```
Request:  FormData { file: File(.docx) }
Response: {
  success, title, characters: CharacterAnalysis[],
  segments: SegmentAnalysis[], meta
}
```

**解析流程：**
1. `mammoth.extractRawText({ buffer })` 提取纯文本
2. 检测有声喵角色定义区：`【cv名】【角色名】【性别】【描述】【出场数】`
3. 解析正文：`【角色名-cv名】"对话内容"` 格式标记
4. 分离旁白和对话，构建结构化数据

### 4.6 POST /api/audiobook/merge

**功能：** 多段音频合并 + 格式转换

```
Request:  {
  segments: [{ audioBase64, duration?, characterName? }],
  format?: 'wav' | 'mp3' | 'm4b'
}
Response: {
  success, audio: string(base64), duration, format, mergedSegments
}
```

**合并逻辑：**
1. 解码base64 → 提取裸PCM
2. 段间插入静音（不同角色0.5s / 同角色0.25s）
3. 拼接 → 添加WAV头
4. mp3/m4b: 调用 `ffmpeg` 转码（`execSync`）

### 4.7 POST /api/audiobook/asr/timestamp

**功能：** ASR语音识别 + 时间戳

```
Request:  { audioBase64: string, mimeType: string }
Response: {
  success, text, timestamps: [{ text, startMs, endMs }],
  duration, lrc, srt
}
```

### 4.8 POST /api/audiobook/voices/clone

**功能：** 声音克隆

```
Request:  { sampleBase64, sampleMimeType, text, emotion?, style?, voice? }
Response: { success, audio: base64, duration, format }
```

**特殊处理：** 指数退避重试（429 rate limit）: 10s → 20s → 40s，最多3次

### 4.9 POST /api/audiobook/voices/design

**功能：** 自然语言音色设计

```
Request:  { description: string, text?: string, optimizeText?: boolean }
Response: { success, audio: base64, duration, format }
```

### 4.10 POST /api/audiobook/voices/polish

**功能：** AI润色音色描述

```
Request:  { description: string }
Response: { success, polished: string }
```

### 4.11 GET /api/audiobook/voices/preset

**功能：** 获取预置音色列表

```
Response: { voices: PresetVoice[], total: number }
```

## 5. 引擎层详解

### 5.1 MiMo TTS 引擎 (mimo-tts.ts)

**API地址：** `https://token-plan-cn.xiaomimimo.com/v1/chat/completions`  
**模型：** `mimo-v2.5-tts`  
**认证：** `api-key` header (`MIMO_API_KEY`)

**8款预置音色：**

| ID | 名称 | 性别 | 风格 |
|----|------|------|------|
| 冰糖 | 冰糖 | 女 | 甜美女声，温暖柔和，适合旁白 |
| 茉莉 | 茉莉 | 女 | 温柔女声，细腻感性 |
| 苏打 | 苏打 | 男 | 阳光男声，清朗明快 |
| 白桦 | 白桦 | 男 | 沉稳男声，浑厚有力 |
| Mia | Mia | 女 | English female, sweet |
| Chloe | Chloe | 女 | English female, gentle |
| Milo | Milo | 男 | English male, warm |
| Dean | Dean | 男 | English male, deep |

**参数传递机制：**
```
emotion → user message
emotionIntensity > 5 → "情感强度X/10" 加入 user message
speed='slow' → "语速缓慢" 加入 user message
speed='fast' → "语速加快" 加入 user message
specialNote → 直接加入 user message（如"低语""气声"）
style → 加入 user message
text → assistant message
```

**情绪标签分类 (EMOTION_TAGS)：**
- basic: 开心/悲伤/愤怒/恐惧/惊讶/兴奋/委屈/平静/冷漠
- complex: 怅然/欣慰/无奈/愧疚/释然/嫉妒/厌倦/忐忑/动情
- tone: 温柔/高冷/活泼/严肃/慵懒/俏皮/深沉/干练/凌厉
- timbre: 磁性/醇厚/清亮/空灵/稚嫩/苍老/甜美/沙哑/醇雅
- style: 夹子音/御姐音/正太音/大叔音/台湾腔
- dialect: 东北话/四川话/河南话/粤语
- character: 孙悟空/林黛玉

### 5.2 MiMo ASR 引擎 (mimo-asr.ts)

**模型：** `mimo-v2.5-asr`

**功能：**
- `transcribe(params)` → 单个音频识别
- `transcribeBatch(params)` → 批量识别
- `generateLRC(timestamps)` → LRC字幕
- `generateSRT(timestamps)` → SRT字幕

### 5.3 MiMo VoiceClone 引擎 (mimo-voice-clone.ts)

**模型：** `mimo-v2.5-tts-voiceclone`

**样本要求：**
- 格式：mp3/wav
- 大小：≤7.5MB
- 建议：清晰、无背景音、10-30秒

### 5.4 MiMo VoiceDesign 引擎 (mimo-voice-design.ts)

**模型：** `mimo-v2.5-tts-voicedesign`

**关键方法：**
- `design(params)` → 设计音色并生成音频
- `preview(description, previewText?)` → 预览
- `getDesignSuggestions(characterType)` → 按角色类型给建议

### 5.5 DOCX 解析器 (docx-parser.ts)

**外部依赖：** `mammoth` (npm包)

**核心函数：**
```typescript
parseDocxBuffer(buffer: Buffer): Promise<DocxParseResult>
parseDocxText(plainText: string): DocxParseResult
```

**有声喵格式识别：**
```
角色定义区：5个方括号 → 【cv名】【角色名】【性别】【描述】【出场数】
正文标记：【角色名-cv名】"对话内容"
分隔线：连续5个以上横线
章节标题：第X章...
```

### 5.6 画本标记解析器 (story-parser.ts)

**核心函数：**
```typescript
parseStoryCharacterDefs(text) → StoryCharacter[]
parseStoryLine(line, knownCharacters?) → StorySegment[]
parseStoryBook(text) → { characters, segments, title }
```

**支持格式：**
- 有声喵格式：`旁白文本【角色名-cv名】"对话"`
- 讯飞格式：`【角色名】对话`
- 通用格式：`角色名："对话"`

### 5.7 段落合并器 (merge-segments.ts)

**核心函数：**
```typescript
mergeConsecutiveSegments(segments, characters) → 合并连续旁白(≤300字)和同角色对话(≤200字)
splitDialoguePrefix(segments) → 从对话中分离"XXX说："为旁白
processAnalysisResult(segments, characters) → 完整管道
```

### 6.2 核心数据流

```
用户输入章节文本
    │
    ▼
[AI分析 / 手动设置 / 导入画本]
    │
    ▼
analysisResult: { characters[], segments[] }
    │
    ├──→ localStorage 缓存 (mojing_analysis_{chapterId})
    │
    ▼
editedCharacters[] + editedSegments[]  ← 用户微调
    │
    ├──→ 撤回/重做栈 (undoStack/redoStack, 最多30层)
    │
    ▼
[生成单段 / 一键生成全部]
    │
    ├──→ POST /api/audiobook/generate
    │        │
    │        ▼
    │    MiMoTTSEngine.generate()
    │        │
    │        ▼
    │    base64 audio → audioCache[segKey]
    │
    ▼
[合并导出]
    │
    ├──→ POST /api/audiobook/merge
    │        │
    │        ▼
    │    WAV拼接 + 可选ffmpeg转码
    │
    ▼
下载文件 (WAV/MP3/M4B)
```

## 8. 数据持久化方案

### 8.1 当前方案

| 数据类型 | 存储位置 | 持久化 |
|----------|----------|--------|
| 项目/章节/角色 | localStorage + Turso | ✅ 持久化 |
| AI分析结果 | localStorage (mojing_analysis_{chapterId}) | ⚠️ 仅浏览器缓存 |
| 已生成音频 | React state (audioCache) | ❌ 内存，刷新丢失 |
| 设计/克隆音色 | React state (designedVoices/clonedVoices) | ❌ 内存，刷新丢失 |
| 画本导入数据 | React state | ❌ 内存，刷新丢失 |

### 8.2 关键发现

⚠️ **有声书核心数据模型（AudiobookProject, AudioChapter, AudioSegment, VoiceProfile）在 `src/lib/audiobook/index.ts` 中只定义了接口，但没有数据库表和store层实现。** 所有有声书数据目前：
- 分析结果: 纯API返回，前端内存持有
- 音频: base64通过API返回，无持久化
- 画本: 纯API解析返回

**这意味着刷新页面后，已生成的音频会丢失，需要重新生成。**

## 一、一条知识条目长什么样

每条条目就是一个 `KnowledgeItem` 对象，加到 `src/lib/ai/knowledge-base.ts` 的 `KNOWLEDGE_BASE` 数组里：

```typescript
{
  id: 'polish_A2_Bclass',        // 唯一ID，见命名规范
  category: 'polish',             // 分类：iron_rules | techniques | polish | genre_params | narrative | operations
  title: 'B类禁用词 — 常规禁用词',  // 人类可读的标题
  content: `【B类禁用词】
词表：突然、觉得、非常、正在、看见、听见、特别、真的
规则：同一自然段内≥3次触发干预
建议：每段不超过2次，交替使用同义表达或直接删`,  // 规则原文
  tags: ['禁用词', 'B类', '突然', '觉得', '非常'],  // 检索关键词
  timing: 'always',               // 注入时机
  priority: 4,                    // 优先级 1-5
}
```

## 四、polish 分类 · 需要补的条目

### 4.1 A-2 禁用词分层（~10条）

把每个禁词级别的规则拆成独立条目，方便按需检索：

- **A类递进判断句**：`不是…是…」「不仅是…更是…」「虽然…但是…」 — 一章最多1次
- **B类常规禁用词**：突然、觉得、非常、正在、看见、听见 — 同段≥3次触发
- **B类扩展**：特别、真的、其实、实际上、完全、绝对、一定 — 密度控制
- **C类连接词L1-L2**：然后、于是、就 — 完全放开
- **C类连接词L3-L5**：然后、于是、就 — 每段≤2次
- **C类语气词**：吧、呢、吗 — 依角色性格保留或删
- **D类事后扫描**："的"连续≥4、动词重复≥3、主语相同、信息复述、意象重复
- **AI高频词·实词限量**：每章核心词<5次
- **AI高频词·删除词**：值得注意、令人、不禁、仿佛、似乎
- **AI高频词·系动词回避**：是、有、在、让、使、令、将、被、把、对
- **禁用句式**：AI对话语、总结语、引导语

### 4.2 工具Q 四级净化体系（~15条）

每级拆成细则条目：

**第一级·词汇替换：**
- 高频副词删除（1条，含词表）
- 生硬词替换（1条，含对照表）
- 机械化动作描写自然化（1条）
- 情绪化修饰词删除（1条）
- 特定意象/比喻形容词替换（1条）

**第二级·句式优化（4条）：**
- 删解释/剧透、删总结性描写、不解释原则、核心动作保留

**第三级·段落重构（5条）：**
- 删铺垫、删无意义对话、删支线、删上帝视角、删重复信息

**第四级·场景适配（3条）：**
- 人设语言匹配、正式/日常区分、开头/正文差异化

### 4.3 工具R 自然表达替换速查表（20条）

每组一条独立条目，方便检索特定替换：

```
R01 突然 → [动作+停顿]或删
R02 觉得 → 感觉或删
R03 非常/很/特别 → 删或用具体程度词
...
R20 某种/一种说不出的 → 具体描写
```

### 4.4 工具AC B-8 精修质检（15条）

每条检查项一条独立条目：

- 的得地检查、中文标点、省略号规范、缩进空格、章节编号
- 句号换行规则L1-L2、句号换行规则L3-L4、句号换行规则L5
- 开篇冲突、视角限制、主角主动性、三位一体、节奏推进
- 白描检查、禁止总结/剧透、删除符号、删除空行
- 防守规则联动（动作后禁解释/精致句分配/感官切换/平庸比喻）

# 墨境质量系统 · 设计方案

> 日期：2026-07-10
> 状态：草案
> 阶段：A — 统一质量中心

---

## 1. 目标

将编辑器右侧栏合规检测、章末自检弹窗、质检仪表盘三个分散的质量触点，整合为统一的「质量中心」体验。每个触点使用用户语言而非技术术语，并统一站点的浅色视觉风格。

## 2. 架构概览

```
质量中心 (统一入口)
├── 实时检测 (右侧栏)    ← paragraphCheck() 8项段落级
├── 章末自检 (弹窗)      ← chapterEndCheck() 23项章节级
└── 质量趋势 (仪表盘)     ← report-store 跨章聚合

底层：
├── compliance.ts      ← 检测引擎（不动）
├── report-store.ts    ← 报告持久化（已有）
└── report-modal.tsx   ← 已重构（需改文案/视觉）
```

## 3. 统一入口

编辑器顶栏右端：

```
[保存] [质量 ▼] [...其他]
         ├─ 实时检测         → 展开右侧栏，切到质量 tab
         ├─ 章末自检         → 弹出完整报告
         └─ 质量趋势         → 跳转 /quality-check/[id]
```

顶部「章末自检」独立按钮保留（常用操作），但右侧栏底部加「完整检测 →」直达弹窗。

## 4. 视觉规范

全局浅色主题，白色卡片 + `border-border/50` + `shadow-sm`。

| 组件 | 风格 | 参考 |
|------|------|------|
| 章末自检弹窗 | `bg-white rounded-[20px] shadow-modal` | 现有弹窗规范 |
| 质量仪表盘 | 白色卡片网格 | `/admin` 风格 |
| 编辑器右侧栏 | 浅色面板，非暗色玻璃态 | 编辑器左侧栏风格 |

编辑器右侧栏改为与左侧栏一致的 `glass-panel border-l border-border` 浅色风格，不再使用暗色透明背景。

## 5. 实时检测（右侧栏·8项）

### 5.1 数据源

调用 `paragraphCheck(text, chapterAUsed)`，取全部 8 项结果。

### 5.2 呈现

每条固定三行：

```
┌─────────────────────────────────────┐
│ ⚠️ 这一段写了太多件事                  │  ← 问题标题（口语化）
│ 主语频繁切换，读者不知道该跟谁。一段   │  ← 解释 + 改法
│ 只聚焦一个人物的动作。                │
│ [跳到第 3 段]                        │  ← 可点击跳转
└─────────────────────────────────────┘
```

### 5.3 8 项文案

| 检测项 | 标题 | 解释 | 改法 |
|--------|------|------|------|
| 单一意图 | 这一段写了太多件事 | 主语频繁切换，读者不知道该跟谁 | 一段只聚焦一个人物的行动 |
| 身体承载 | 情绪是"说"出来的，不是"演"出来的 | "他感到害怕"不如"他手心全是汗" | 用身体动作代替心理描写 |
| 禁用词混入 | 有几个词连续出现太多次 | 同一段里某些词反复出现，像卡带 | 换个说法或删掉多余的 |
| 动作后解释 | 动作后面跟了解释 | 做了就做了，解释反而削弱冲击力 | 删掉"因为""所以"，只留动作 |
| 视角越界 | 不小心写到别人心里去了 | 当前视角角色不可能知道别人在想什么 | 删掉其他角色的内心描写 |
| 大白话测试 | 这句有点绕，读出声试试 | 太书面化的句子打断阅读节奏 | 改成口语化短句 |
| 精致句位置 | 好句子放错了地方 | 精彩描写要放在情绪高潮 | 移到动作或情感爆发点 |
| 生造比喻 | 这个比喻有点勉强 | 比喻要来自读者熟悉的日常经验 | 换一个立刻能懂的意象 |

### 5.4 颜色规则

- 🔴 阻断项（禁用词、动作后解释）— 红底，强制提醒
- 🟡 其他 6 项 — 黄/灰色，仅提示不强制
- 默认折叠，只显示标题行

## 6. 章末自检（弹窗·23项）

### 6.1 数据源

调用 `chapterEndCheck(text, genre, chapterAUsed)`，取全部 23 项结果。

### 6.2 布局

```
┌─ 头部 ──────────────────────────────┐
│ 章末自检报告              评分 4/5   │
│ 3,248字 · 身体密度42% 开头有钩子 ✓  │
├─ 一句话评价 ────────────────────────┤
│ 总体不错！开头悬念到位。对话占比略低  │
│ 建议加一段对手戏。                   │
├─ 待处理 ───────────────────────────┤
│ 🔴 2处动作后紧跟解释                 │
│ 🟡 对话占比偏低(18%)                │
│ 🟡 连续5段紧张无透气                │
├─ 需人工判断 ───────────────────────┤
│ ○ 视角一致性  [🤖 AI分析 →]        │
│ ○ 时间线一致性 [🤖 AI分析 →]        │
├─ 底部 ─────────────────────────────┤
│ [返回修改] [重新检测] [通过并保存]    │
└─────────────────────────────────────┘
```

### 6.3 一句话评价生成

由评分 + 关键指标拼接得出，例如：
- 4-5 分 + 合规：`"总体不错！{正面指标}，{可优化项}。"`
- 3 分：`"有几个地方需要注意：{主要问题}。"`
- 1-2 分：`"这一章存在明显问题，建议修改后重新检测。"`

### 6.4 三层分类

| 分类 | 颜色 | 说明 |
|------|------|------|
| 待处理 | 🔴🟡 | 程序可精确检测的问题（带改法建议） |
| 需人工判断 | ○ | 程序无法自动化，标灰 + AI 分析入口 |
| （已通过项默认折叠） | ✓ | 不占视觉空间 |

## 7. 质量仪表盘（`/quality-check/[id]`）

### 7.1 数据源

`getProjectQualitySummary(projectId)` — 聚合所有已保存报告。

### 7.2 布局

```
┌─ 一句话总评 ────────────────────────┐
│ 前半段质量稳定，第6章开始身体密度偏低  │
├─ 概要卡片 4 枚 ────────────────────┤
│ 📈4.2均分  ✓80%合规  📝3.2万字  📊8章│
├─ 图表 3 格 ────────────────────────┤
│ 评分趋势    身体密度    字数分布      │
├─ 各章一览 ─────────────────────────┤
│ 第1章 ████████████░░ 4分 ✓         │
│ 第2章 ██████░░░░░░░░ 3分 ⚠️         │
│ ...点击行展开详细报告                │
└─────────────────────────────────────┘
```

### 7.3 各章一览

用进度条代替表格，长度 = 评分/5，颜色绿/黄/红。右侧标违规类型。点击行展开该章节的完整 23 项报告。

## 8. 分阶段计划

| 阶段 | 内容 | 依赖技能包 |
|------|------|-----------|
| **A** — 统一质量中心 | 入口合并、右侧栏8项升级、文案改写、视觉统一 | 本次范围 |
| **B** — AI 深度分析 | 12项空壳接入 AI API，编辑器 AI 建议面板 | `claude-api` |
| **C** — 商业级看板 | 多作品对比、PDF 导出、质量目标追踪 | `api-and-interface-design` |

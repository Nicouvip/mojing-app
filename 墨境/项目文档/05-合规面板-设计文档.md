# 05 · 编辑器合规面板 · 设计文档

> 版本：v1.0 | 日期：2026-07-08 | 状态：设计已完成

---

## 一、背景

墨境编辑器中需要一个实时合规检测面板，让作者在写作过程中能直观地看到：
- 身体密度、冲突强度等核心指标
- 禁用词、动作后解释等合规状态
- 冷却系统、感官通道等引擎状态
- 伏笔追踪、角色在场等创作信息

这些数据墨境的 AI 引擎层已有计算能力（`ai/compliance.ts` 等），但缺少可视化展示。

---

## 二、设计方案

### 2.1 交互结构（三层折叠）

```
收起状态：
  编辑器右侧边缘 → 一个铜金小标签（ChartBar图标）
                  hover 时显示 tooltip「合规面板」

展开第一层（点击标签）：
  滑出 300px 宽面板 → 四个分类标题 + 关键数字
  ┌──────────────────────────┐
  │ 📊 核心指标     41% L3   │
  │ ⚠️ 合规检测      2项     │
  │ 🎭 创作引擎      S1◐    │
  │ 📋 伏笔与角色    3条     │
  └──────────────────────────┘

展开第二层（点击某个分类）：
  该分类内展开详细数据，其他分类保持折叠
  每个分类独立展开/收起，互不影响
```

### 2.2 四个分类的详细数据

| 分类 | 一级展示 | 二级展开 |
|------|---------|---------|
| **核心指标** | 身体密度 41% + 冲突强度 L3 | 身体密度(进度条)、冲突强度(进度条)、字数、对话占比 |
| **合规检测** | 违规项数 2 | A类递进判断句(绿点/红点)、B类禁用词、C类连接词、动作后解释 |
| **创作引擎** | 场景冷却 S1●S2○S3◐ | 六场景方法状态、五感通道状态、当前风格识别 |
| **伏笔与角色** | 活跃伏笔 3条 | 伏笔列表(◉标识)、角色在场数 |

### 2.3 交互规则

- 点击右侧标签 → 面板从右滑入/滑出（300ms cubic-bezier）
- 点击分类标题 → 展开/收起该分类详情（独立控制）
- 点击面板头部 ✕ → 收起面板
- 点击面板外部 → 收起面板
- 按 ESC 键 → 收起面板
- hover 标签 → 显示 tooltip「合规面板」

### 2.4 视觉规范

| 项目 | 取值 |
|------|------|
| 面板底色 | `rgba(245,242,237,0.97)` + backdrop-filter 毛玻璃 |
| 品牌色 | `#c4956a`（铜金） |
| 文字色 | `#1a1814`（暖黑） |
| 警告色 | `#b06060` |
| 通过色 | `#6a8a6a` |
| 警告黄 | `#b8a060` |
| 面板宽 | 300px |
| 分类圆角 | 10px |
| 面板左侧圆角 | 12px |
| 图标风格 | Phosphor 线性风格，stroke-width 1.5，round cap/join |
| UI字体 | Inter |
| 编辑器字体 | Noto Serif SC |

## 三、组件结构

### 3.1 前端组件树

```
EditorPage
├── Editor (编辑器主体)
│   ├── TipTapEditor
│   └── EditorToolbar
├── CompliancePanel (合规面板)        ← 新增
│   ├── PanelTab                      ← 收起标签
│   ├── PanelContainer                ← 滑出面板
│   │   ├── PanelHeader               ← 面板头部（标题+关闭）
│   │   ├── CategoryGroup (×4)        ← 四个分类
│   │   │   ├── CategoryHeader        ← 分类标题栏（图标+名称+数字+展开箭头）
│   │   │   └── CategoryDetail        ← 展开详情（进度条/列表/状态）
│   │   └── ...
```

### 3.2 数据流

```
complianceEngine (引擎层)
    ↓ 实时计算
ComplianceData (数据结构)
    ├── metrics: { bodyDensity, conflictLevel, wordCount, dialoguePercent }
    ├── compliance: { classA, classB, classC, postActionExplain }
    ├── engine: { scenes, senses, style }
    └── story: { foreshadowing, charactersInScene }
    ↓ props
CompliancePanel (UI组件)
```

### 3.3 数据结构

```typescript
interface ComplianceData {
  metrics: {
    bodyDensity: number        // 0-100
    conflictLevel: 1 | 2 | 3 | 4 | 5
    wordCount: number
    dialoguePercent: number    // 0-100
  }
  compliance: {
    classA: { count: number; max: number }      // 递进判断句
    classB: { violations: number }               // 常规禁用词
    classC: { count: number }                    // 连接词
    postActionExplain: { count: number }         // 动作后解释
  }
  engine: {
    scenes: Array<{ id: string; status: 'available' | 'unused' | 'cooling' | 'disabled' }>
    senses: Array<{ id: string; active: boolean }>
    style: string
  }
  story: {
    activeForeshadowing: Array<{ name: string; status: string }>
    charactersInScene: number
    totalCharacters: number
  }
}
```

## 四、验收标准

- [ ] 编辑器右侧有收起/展开标签
- [ ] 点击标签面板以动画形式滑入/滑出
- [ ] 面板显示四个分类（核心指标/合规检测/创作引擎/伏笔与角色）
- [ ] 每个分类显示对应关键数字
- [ ] 点击分类展开详细数据，再点击收起
- [ ] 各分类独立展开/收起
- [ ] 点击 ✕ 或面板外或按 ESC 收起面板
- [ ] hover 标签显示 tooltip
- [ ] 颜色/字体/间距与品牌规范一致
- [ ] 图标统一为 Phosphor 风格线性 SVG
- [ ] 响应式：桌面正常显示，移动端隐藏或改为全屏
- [ ] 无障碍：键盘可操作、aria 标签完整
- [ ] `prefers-reduced-motion` 关闭动画

# 06 · 四阶段工作流切换 · 设计文档

> 版本：v1.0 | 日期：2026-07-08 | 状态：设计已确认

---

## 一、背景

墨境编辑器目前只有"写作"一个模式，但张总的 V10.0.2 核心引擎定义了四阶段工作流（规划→写作→自查→交付）。本功能在编辑器底部加入工作流状态栏，让编辑器跟随写作进度切换展示内容。

---

## 二、设计方案

### 2.1 交互结构

编辑器底部一条工作流状态栏：

```
┌──────────────────────────────────────────────────────────────┐
│ ✓规划     →     ● 写作     →     自查     →     交付        │
│                 字: 2,400 | 密度: 41% | L3                   │
│         [续写] [润色] [扩写] [合规检测] [灵感]              │
└──────────────────────────────────────────────────────────────┘
```

- 四个阶段横排，已完成显示 ✓，当前阶段铜金高亮，未到置灰
- 点击任意阶段可手动切换
- 底部工具按钮随阶段变化

### 2.2 各阶段行为

| 阶段 | 编辑器变化 | 底部按钮 |
|------|-----------|---------|
| **规划** | 左侧栏冷却矩阵展开 + 大纲概览 | [生成规划] [选场景技法] |
| **写作** | 正常编辑 + 合规面板 | [续写] [润色] [扩写] [灵感] |
| **自查** | 合规面板自动展开 + 违规文中高亮 | [一键检测] [替换建议] |
| **交付** | 底部弹出检测报告 | [确认交付] [导出状态] |

### 2.3 流程引导

- 规划→写作：点「生成规划」后自动切换
- 写作→自查：点合规检测按钮时建议切换
- 自查→交付：检测没问题时自动引导
- 交付完成：本章完成，提示下一章

### 2.4 视觉规范

配色、字体与已有品牌一致（暖白底、铜金品牌色）。

---

## 三、组件结构

```
WorkflowBar（底部工作流状态栏）
├── StageButton（×4）：规划/写作/自查/交付
│   └── 状态：current | done | pending
├── MetricsBar：字数/身体密度/冲突强度
└── ToolButtons：随阶段变化的工具按钮
```
